## 현재 구현 상태

현재 `wallet-service`에서 문서화 가능한 API는 인증 영역이 중심입니다.

- 회원가입
- 로그인
- 로그아웃
- Access Token 재발급

아래 영역은 아직 본격 구현 전이거나 문서화할 만큼 고정되지 않았습니다.

- 사용자 정보 조회/수정
- 지갑 생성/조회
- 포인트 충전/사용/취소
- 거래내역 조회
- 관리자 API

다음 단계로는 인증 API 문서 형식을 그대로 가져가서 사용자, 지갑, 포인트 API 문서를 이어서 확장하면 됩니다.
